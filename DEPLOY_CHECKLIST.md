## 1. 지금 바로 배포 가능 (정적 호스팅)

이 사이트는 **순수 정적 파일**(HTML/CSS/JS)이라 Vercel·Netlify·GitHub Pages·
부산시 웹서버 어디든 폴더째 올리면 바로 동작합니다. 빌드 과정이 없습니다.

올릴 파일:
```
index.html  register.html  archive.html  admin.html
image-slot.js
css/        (styles.css, admin.css)
js/         (config.js, api.js, site.js, main.js, admin.js, reserve.js)
images/     (keyvisual.jpg)
```
문서(README, *_SETUP.md)와 screenshots/ 폴더는 올리지 않아도 됩니다.

### 외부 의존성 (모두 HTTPS 공개 CDN — 키 불필요)
- Google Fonts (Playfair Display, Bodoni Moda, Marcellus, Space Mono)
- Pretendard (jsdelivr)
- qrcode-generator (예약 QR 발급)
- html5-qrcode (관리자 현장 스캔)

> 인터넷이 차단된 폐쇄망에 올릴 경우 위 CDN 파일을 로컬로 받아 경로를 바꿔야 합니다.
> 일반 웹 배포라면 그대로 두면 됩니다.

## 2. 점검 완료 항목 ✓

- 4개 페이지(메인·예약·아카이브·관리자) 콘솔 에러 **없음**
- 깨진 `#` 링크 정리 (푸터 死링크 제거, 정적 더미 마크업 제거)
- 한국어 줄바꿈(어절 단위) + 문장 단위 줄바꿈 정리
- 가짜 수치 제거 — 실제 확정값만 표시 (12쇼·3일·300석·3B홀)
- 빈 섹션은 모두 "준비중입니다."로 통일
- IR(투자매칭) 섹션 숨김 + 섹션 번호 자동 재정렬
- 관리자 13개 탭 ↔ 패널 정상 매칭
- 예약 페이지 로컬 모드에서 12쇼·선착순·QR 발급 정상

## 3. 배포 후 채워야 할 콘텐츠 (관리자에서)

관리자(`admin.html`)에서 코드 수정 없이 입력:

| 항목 | 위치 | 비고 |
|---|---|---|
| 키비주얼·오프닝 이미지 | 메인 곳곳 image-slot | 드래그 업로드 |
| 참여 브랜드 로고 | 참여 브랜드 탭 | 확정 후 등록 (현재 "준비중") |
| 참여 대학 로고 | 참여 대학 탭 | 공모 후 등록 |
| 아카이브(역대 행사) | 아카이브 탭 | 연도·제목·이미지·링크 |
| 언론 보도 | 언론 보도 탭 | 수동 등록 즉시 가능 |
| 하이라이트 영상 | On Film 영상 탭 | 행사 후 (현재 "준비중") |

## 4. 키 입력이 필요한 연동 (선택)

각 기능은 키가 없으면 **"준비중"/플레이스홀더**로 안전하게 표시되고,
키를 넣으면 자동 활성화됩니다.

| 연동 | 설정 위치 | 문서 |
|---|---|---|
| **관람 예약 실서버** (선착순 정원 공유) | `js/config.js`의 `SUPABASE` | `SUPABASE_SETUP.md` |
| 예약 확인 문자/알림톡 | Supabase Edge Function | `SUPABASE_SETUP.md` §8 |
| 언론 보도 자동 수집 | 관리자 → 언론 보도 → 프록시 URL | `PRESS_SETUP.md` |
| 인스타그램 자동 피드 | `js/config.js`의 `instagram.token` | IG Graph API |
| 네이버 지도 | 관리자 → 오시는 길 → Client ID | NAVER Cloud Maps |

> **중요 — 관람 예약**: 현재는 브라우저 저장(localStorage) 기반이라 한 기기 안에서만
> 동작합니다. 실제로 여러 관람객이 같은 잔여석을 공유하며 선착순 마감되게 하려면
> `SUPABASE_SETUP.md`대로 백엔드를 구축하고 `SUPABASE` 키를 입력해야 합니다.
> 키 입력 후 관리자 → 쇼 관리 → "메인 페이지에 예약 공개"를 켜면 메인에 예약 버튼이 노출됩니다.

## 5. 오픈 시퀀스 (권장 순서)

1. **사전 공개**: 일정·장소·About만 노출 (예약 버튼 숨김 = 현재 상태)
2. 브랜드/대학 확정 → 관리자에서 로고 등록 → 해당 섹션 자동 채워짐
3. 예약 백엔드 구축 + 키 입력 → "예약 공개" ON → 선착순 접수 시작
4. 행사 중: 관리자 → 현장 체크인(스태프 로그인 + QR 스캔)으로 입장 관리
5. 행사 후: 하이라이트 영상·아카이브 등록

## 6. 관리자 접근 보안 (배포 전 권장)

`admin.html`은 현재 URL만 알면 접근 가능합니다. 실배포 시:
- 호스팅의 비밀번호 보호(Basic Auth) 또는 비공개 경로 적용 권장
- 또는 Supabase 스태프 로그인을 관리자 전체에 확대 적용
- 푸터의 "Admin" 링크는 운영 중 제거하는 것을 권장 (`index.html` 최하단)
